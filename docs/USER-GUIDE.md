## Overview

SignatureHub provides a complete electronic signature solution with:

- **Multi-party signing packages** - Collect signatures from multiple people on one document
- **Role-based signing** - Signers are assigned roles (participant, guardian, trainer, etc.)
- **Identity verification** - Email or SMS verification before signing
- **Real-time status tracking** - Public status pages to share progress
- **Admin notifications** - Emails when packages are created and completed

## Context Fields

Context fields display relevant information (event name, horse, rider, etc.) when provided in the merge variables. They appear in:

1. **Signing Page** - In the document review header
2. **Emails** - In a highlighted box below the document name
3. **Status Page** - Below the package code

### Supported Context Fields

| Merge Variable Key | Display Label |
|-------------------|---------------|
| `eventName` | Event |
| `eventDate` | Event Date |
| `horseName` | Horse |
| `riderName` | Rider |
| `trainerName` | Trainer |
| `ownerName` | Owner |
| `competitionName` | Competition |
| `venueName` | Venue |
| `className` | Class |
| `divisionName` | Division |

### Example API Request with Context

```json
{
  "templateCode": "ENTRY_AGREEMENT_2024",
  "documentName": "Spring Horse Trials - Entry Agreement",
  "mergeVariables": {
    "eventName": "Spring Horse Trials 2024",
    "eventDate": "June 15, 2024",
    "horseName": "Midnight Star",
    "riderName": "Emily Johnson",
    "competitionLevel": "Training Level"
  },
  "signers": [...]
}
```

## URLs Reference

| Page | URL Pattern | Access |
|------|-------------|--------|
| Signing Page | `/sign/:token` | Private (unique per signer) |
| Status Page | `/status/:packageCode` | Public (shareable) |
| Replace Signer | `/replace/:token` | Private (admin only) |
| Health Check | `/health` | Public |
